# DeepSeek Prompt 摘录

本文档展示 Boss 直聘招呼语生成器中使用 DeepSeek API 生成招呼语的完整 prompt 设计。

---

## 设计意图

**场景**：根据 Qwen 提取的结构化 JD 信息 + 用户简历，生成针对 Boss 直聘的打招呼语。

**核心设计思路**：
1. 将 JD 结构化信息和简历合并输入，让模型自行匹配
2. 动态调整第三段的写作角度，根据岗位类型（ib/invest/ai/general）变化
3. 通过 `confidence` 控制表达的保守程度
4. 输出 JSON 格式，键为 `greeting`，只包含招呼语文本

---

## Prompt 结构

**调用位置**：`src/lib/ai.js` → `generateGreetings()` (行 261-304)

### System Prompt

```markdown
你是一个擅长求职对话文案的助手，只输出严格 JSON。
```

### User Prompt（核心生成逻辑）

```markdown
你是一个求职沟通文案助手。请根据"岗位结构化信息"和"候选人简历信息"，生成适合Boss直聘使用的打招呼语。

目标：
让招聘方快速感受到候选人与岗位的匹配度，并愿意回复。

生成要求：
1. 风格自然，像真实求职者，不像AI，不像正式邮件。
2. 不要使用空泛套话，如"非常荣幸""期待加入"等。
3. 优先围绕岗位字段中的 best_greeting_angle 来组织表达。
4. 结合候选人真实经历和岗位关键词来写，不要脱离简历乱发挥。
5. 如果岗位要求与简历并非完全匹配，就突出相近经验、可迁移能力或结果导向。
6. 如果 confidence 低于 60，不要过度引用具体细节，优先写更稳妥、更泛化的表达。
7. 如果 company_name 或 job_title 明确，可以自然带入；如果为空，则不要硬写。
8. 输出 1 条质量最高的招呼语，控制在 100-150 字，适合Boss直聘聊天场景。
9. 只输出严格 JSON，不要解释，不要加 markdown。

话术结构规范：
- 开头：用礼貌、真诚的语气打招呼，表达对岗位或公司的兴趣，如"您好！我对贵公司xx岗位非常感兴趣，很期待与您进一步交流。"
- 第一段：简要介绍学历背景（院校+专业）和职业经历（公司+岗位+年限），让对方快速建立基本认知。
- 第二段：重点介绍投行/主要职业经历中积累的核心能力，结合岗位需求选取最相关的角度。
{第三段动态内容}
- 结尾：礼貌、积极地表达进一步交流的意向，如"非常期待有机会进一步沟通！"

岗位结构化信息：
{jobFields JSON}

{resumeSection}

{exampleSection}

输出格式：
{
  "greeting": ""
}
```

## 第三段动态指令（核心特色）

**调用位置**：`src/lib/ai.js` → `buildThirdParagraphInstruction()` (行 248-259)

根据 `inferJobType()` 判断的岗位类型，动态插入第三段的内容侧重：

### 岗位类型判断逻辑

```javascript
// inferJobType() 使用关键词匹配判断
ibKeywords: ['投行', 'ipo', '承销', '保荐', '上市', '尽职调查', '再融资', '定增', '配股', '债券发行']
investKeywords: ['投资', '投研', '融资', '并购', '基金', '金融', '券商', '行业研究', '估值', 'pe', 'vc']
aiKeywords: ['ai', '模型', '大模型', '算法', '产品', 'agent', '策略', '运营', '增长', '数据', '平台', '智能']
// 其他 -> general
```

### 动态插入的第三段指令

| 岗位类型 | 第三段指令 | 理由 |
|----------|-----------|------|
| `ai` | 穿插 AI 工具的实际使用经验，强调真实落地的项目和效率提升成果 | 展示实际动手能力而非浅层体验 |
| `ib` | 点出投行项目中承担的具体职责（财务尽调、申报文件撰写、募投方案设计等），用项目名称或规模数据支撑 | 投行强调项目经验和可交付成果 |
| `invest` | 点出数据分析、估值建模或行业研究方面的具体成果，用数字或案例支撑 | 投资强调研究能力和量化成果 |
| `general` | 突出简历中最相关的一项核心工作成果或可迁移能力，用具体事例说明 | 泛化处理，稳妥为主 |

---

## 调用示例

```javascript
const thirdParagraph = buildThirdParagraphInstruction(jobType);
// jobType: 'ib' | 'invest' | 'ai' | 'general'

const userContent = `你是一个求职沟通文案助手。请根据"岗位结构化信息"和"候选人简历信息"...
...
${thirdParagraph}
...`;

const raw = await callDeepSeek(
  '你是一个擅长求职对话文案的助手，只输出严格 JSON。',
  userContent,
  600
);

const parsed = JSON.parse(raw.match(/\{[\s\S]*\}/)?.[0]);
return parsed.greeting;
```

## 生成效果控制

通过 JD 的 `confidence` 字段控制输出保守度：

| confidence | 策略 | 示例 |
|------------|------|------|
| > 60 | 可引用具体细节 | "在字节跳动主导过 DAU 500万 的策略项目" |
| < 60 | 更泛化表达 | "在头部互联网公司有过相关经验" |

## 话术结构速查

```
[开头] 礼貌问候 + 表达兴趣
  ↓
[第一段] 学历背景（院校+专业）+ 职业经历（公司+岗位+年限）
  ↓
[第二段] 核心能力 + 结合岗位需求
  ↓
[第三段] 根据岗位类型动态调整（见上表）
  ↓
[结尾] 积极表达进一步交流意向
```
